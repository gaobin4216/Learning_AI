## 二、架构对比

### 2.1 DeepSeek-V4 系列

| 特性 | 说明 |
|------|------|
| **注意力机制** | 混合注意力：Compressed Sparse Attention (CSA) + Heavily Compressed Attention (HCA) |
| **KV Cache 优化** | 相比 V3.2，1M 上下文仅需 10% KV cache |
| **推理效率** | 单 token 推理 FLOPs 仅为 V3.2 的 27% |
| **残差连接** | Manifold-Constrained Hyper-Connections (mHC) |
| **优化器** | Muon Optimizer（更快收敛、更稳定训练） |
| **权重精度** | FP4 + FP8 混合精度（MoE 专家参数用 FP4，其余用 FP8） |
| **后训练** | 两阶段：领域专家独立培养 (SFT + RL/GRPO) → 统一整合（on-policy distillation） |

### 2.2 MiMo-V2.5 系列

| 特性 | 说明 |
|------|------|
| **注意力机制** | 混合注意力：Sliding Window Attention (SWA) + Global Attention (GA)，6:1 比例 |
| **KV Cache 优化** | 通过可学习的 attention sink bias 减少约 7x KV cache |
| **滑动窗口** | 128 tokens 窗口大小 |
| **多 Token 预测** | 3 个轻量 MTP 模块（dense FFN），输出速度提升 3 倍 |
| **GQA** | 128 Query Heads / 8 KV Heads |
| **专家路由** | 384 个路由专家，每 token 激活 8 个 |
| **后训练** | SFT → 领域专用 RL → Multi-Teacher On-Policy Distillation (MOPD) |
| **多模态支持** | V2.5 版本支持文本/图像/视频/音频（729M ViT + 261M Audio Encoder） |

### 2.3 GPT-5.4 系列

| 特性 | 说明 |
|------|------|
| **模型定位** | OpenAI 面向专业工作、复杂推理、编码和智能体任务的前沿闭源模型 |
| **推理能力** | GPT-5.4 Thinking 强化长任务推理、计划生成、复杂问题分解和上下文保持 |
| **编码能力** | 融合 GPT-5.3-Codex 的编码能力，适合长周期软件工程、调试、前端开发和工具调用 |
| **Agent 能力** | 原生强化计算机使用能力，可结合浏览器、桌面、API、文件和外部工具完成多步工作流 |
| **视觉能力** | 支持更高保真图像输入细节，提升截图理解、文档解析、定位和点击准确性 |
| **开放性** | 不开放权重，主要通过 ChatGPT、OpenAI API、Codex 和企业产品使用 |

## 三、推理模式对比

### DeepSeek-V4 推理模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **Non-think** | 直接输出，无推理过程 | 日常简单任务 |
| **Think High** | `<think>...</think>` 推理链 | 复杂问题分析 |
| **Think Max** | 最大推理深度，需 ≥384K 上下文 | 极端复杂推理 |

### MiMo-V2.5 推理模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **标准推理** | 默认推理模式 | 日常问答、写作、简单代码 |
| **深度推理** | 扩展推理链处理复杂问题 | 数学、逻辑、复杂规划 |
| **Agent 模式** | 专门优化的工具调用和多轮对话能力 | 自动化任务、工具调用、长流程执行 |

### GPT-5.4 推理模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **GPT-5.4 Thinking** | 面向复杂任务的主力推理模式，可先给出工作计划并在长任务中保持上下文 | 研究、写作、编程、数据分析、文档/表格/演示处理 |
| **GPT-5.4 Pro** | 面向最高性能需求的增强版本 | 高难度推理、法律/金融/科研等高复杂度专业任务 |
| **API reasoning effort** | 开发者可按任务调整推理强度和成本 | 产品集成、Agent、自动化工作流 |

## 八、选型建议

### 选择 DeepSeek-V4 的场景

- 需要**最强代码生成和竞赛编程**能力（Codeforces 3206）
- 需要**极致长上下文**稳定性（1M 全程无衰减）
- 需要更成熟的**开源社区生态和部署支持**
- 对**数学推理**要求不是极端苛刻
- 希望控制推理成本，或在本地/私有环境中部署模型

### 选择 MiMo-V2.5 的场景

- 需要**多模态**能力（图像/视频/音频理解）
- 需要**最强数学推理**（GSM8K 99.6, MATH 86.2）
- 需要**更快的输出速度**（MTP 3x 加速）
- 偏好小米生态系统集成
- 希望使用开放权重，并保留本地部署和二次开发空间

### 选择 GPT-5.4 的场景

- 需要**最成熟的通用助手体验**，包括写作、学习、编程、研究和办公
- 需要**复杂专业工作流**，例如表格、文档、演示、网页研究、代码仓库和多工具协作
- 需要**强 Agent 能力**，让模型操作浏览器、桌面、文件、API 或开发环境
- 需要**企业级稳定性、产品生态和安全治理**
- 不需要本地部署权重，可以接受 API 或 ChatGPT 的商业使用方式

### 三者共同的优势

- 都强调**长上下文能力**，适合处理长文档、长代码和复杂多轮任务
- 都具备较强的**推理、代码和工具调用潜力**
- 都在向**Agent 化工作流**发展，不只是回答问题，而是执行任务

### DeepSeek 和 MiMo 相对 GPT 的共同优势

- 开放权重，适合**私有化部署、本地推理和二次开发**
- 成本结构更可控，适合大规模定制化应用
- 可结合 vLLM、SGLang、Transformers 等开源推理框架自由优化

### GPT 相对 DeepSeek 和 MiMo 的主要优势

- 产品化程度更高，ChatGPT、API、Codex 和企业生态完整
- 在真实办公、编程、工具调用、浏览器/桌面操作等端到端任务中更成熟
- 闭源模型虽然不可本地部署，但可获得持续更新、统一接口和更完善的安全治理
